# TRD 模板

> 使用说明：完成事实表和判断区后，按此模板撰写 TRD。
> 模板支持 **Simple** 和 **Complex** 两种模式，根据需求复杂度选择合适深度。
> 带有 `<!-- COMPLEX ONLY -->` 标记的章节仅在 Complex 模式下填写。
> 每个 `[ ]` 内为填写指引，填写后删除。

---

# TRD：[需求名称]

| 元数据 | |
|--------|------|
| **关联 PRD** | [链接/文件名] |
| **撰写日期** | YYYY-MM-DD |
| **复杂度** | Simple / Complex |
| **状态** | Draft → Confirmed → In Progress → Completed → Archived |
| **代码依据** | 参见下方 [第2节：现有代码依据] 和附录事实表 |

> **状态生命周期：**
> - **Draft** — 撰写中
> - **Confirmed** — 设计已确认，可进入开发
> - **In Progress** — 开发进行中
> - **Completed** — 功能已上线
> - **Archived** — 归档至 `docs/archive/`
